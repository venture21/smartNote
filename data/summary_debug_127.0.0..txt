### 휴머노이드 제어의 복잡성과 기술적 한계
* 걷는 문제는 상당 부분 해결되었으나, 손의 움직임이 추가되면 수학적 차원(자유도)이 급격히 증가하여 제어가 매우 어려워짐. [cite: 0, 2, 7, 8]
* 손가락 하나의 자유도가 3~4에 달해 팔, 다리와 결합하면 전체 자유도가 너무 커져 현재의 강화 학습 기술로는 한꺼번에 최적화하기 곤란함. [cite: 2, 3]
* 걷기와 스마트폰 조작을 동시에 하는 것과 같은 복합적인 동작은 현재 기술로는 구현이 불가능하며, 각 동작을 분리해서 처리해야 함. [cite: 0, 8, 9]
* 현재 로봇은 시각 지능(보고 판단하는 AI)과 보행 지능(걷는 AI)이 분리되어 있어 비효율적이며, 이를 통합하는 시각 일체형 보행 기술 연구가 필요함. [cite: 4, 5]
* 현재 AI 기술은 다중 접촉을 동반하는 안정적인 전신 작업을 수행하는 데 한계가 있으며, 이를 극복하기 위해서는 컴퓨팅 파워의 비약적인 발전이나 강화 학습의 근본적인 기술 혁신이 필요함. [cite: 3, 4, 8, 10]

### 로봇 손의 기계적 구조와 구현 방식
* 인간의 뇌는 상당 부분을 손과 시각 정보 처리에 사용하며, 이는 휴머노이드에게 손 제어가 얼마나 어려운 과제인지를 시사함. [cite: 11, 12]
* 로봇 손의 구현 방식은 크게 세 가지로 나뉨: 모터를 각 관절에 직접 연결하는 '모터 다이렉트 드라이븐', 힘줄을 모사한 '텐던 드라이븐', 힘줄 대신 링크 구조를 사용하는 '링키지 드라이븐' 방식이 있음. [cite: 13]
* '모터 다이렉트 드라이븐' 방식은 제어가 직관적이지만 모터 소형화의 한계로 부피가 커지는 단점이 있으며, 현재 이 기술로 사과를 잡거나 원기둥을 구멍에 끼우는 작업이 가능함. [cite: 13, 14, 15]
* '링키지 드라이븐' 방식은 모터를 손바닥에 집중시켜 손을 작게 만들 수 있으나, 구조가 복잡하고 정밀 제어가 어려움. [cite: 17, 18, 19]
* 현재 기술로 계란 잡기, 분무기 사용, 가위질, 핀셋 작업 등 다양한 도구 사용이 시연되었으나, 이는 특정 동작을 프로그래밍한 결과이며 AI 기반의 자율적 수행은 아님. [cite: 19, 20, 21]
* 휴머노이드가 인간 사회에서 널리 쓰이는 도구를 사용하기 위해서는 사람과 비슷한 크기의 손이 필수적임. [cite: 16, 17]

### 휴머노이드 손 제어를 위한 AI 학습 방법론
* 손을 사용하는 작업은 자유도가 높고 '실패'를 정의하기 애매하여 강화 학습 적용이 매우 어려움. [cite: 23]
* 이 때문에 손 제어에는 행동뿐만 아니라 손에 느껴지는 감각까지 모방하는 '모방 학습'이 주로 연구되나, 이 역시 감각 센서의 부재 및 데이터 처리 문제로 어려움을 겪고 있음. [cite: 23, 24]
* 현재는 특정 작업을 위해 프로그래밍된 수준이며, 다양한 일을 스스로 판단하여 수행하는 범용적인 손 제어 AI는 아직 개발되지 않음. [cite: 19, 22, 24]
* 향후 과제로는 고차원 공간을 극복할 수 있는 강화 학습 기술 개발, 시각 정보 기반의 손 작업 AI, 힘 제어 기반의 모방 학습 등이 있음. [cite: 25]

### 촉각 센서 기술의 중요성과 발전 방향
* 로봇이 정교한 작업을 수행하고 안전성을 확보하기 위해서는 촉각을 포함한 다중 감각 기술이 필수적임. [cite: 4, 27, 50, 51]
* 현재는 물체를 잡았을 때의 수직 압력(수직 항력) 정도를 측정하는 수준이나, 온도, 떨림, 미끄러짐 등 복합적인 정보를 감지하는 기술이 필요함. [cite: 26, 27]
* 사람의 피부처럼 말랑말랑한 유연 재질 위에 감각 센서를 구현해야 하며, 이로 인해 발생하는 내구성 문제를 해결하기 위해 '자가 치유(self-healing)' 소재 기술이 연구되고 있음. [cite: 27, 28, 29]
* 자가 치유 기술은 화학적 결합이 끊어져도 스스로 복원되는 소재로, 이를 통해 센서의 내구성을 높일 수 있음. [cite: 28, 29, 30]
* 또한, 관절이 접힐 때 회로가 끊어지지 않도록 신축성 있는 회로 기술도 필요하며, 이러한 감각 센서를 전신에 부착하면 협소 공간 이동 시 안전성을 크게 향상시킬 수 있음. [cite: 28, 30, 31]

### 휴머노이드 두뇌 '대뇌-소뇌' AI 모델
* 휴머노이드의 두뇌는 인간의 뇌 구조를 모사한 '대뇌-소뇌' 모델로 개발됨. [cite: 32]
* '대뇌'는 언어 모델(LLM)을 기반으로 하며, 공간, 관계, 절차, 상호작용 등 여러 전문 지식을 가진 작은 AI 모델들의 협업을 통해 추상적 사고와 판단을 담당함. [cite: 33, 34]
* '소뇌'는 생성 AI 모델을 기반으로 하며, 대뇌의 명령과 시각, 촉각 등 감각 정보를 종합하여 실제 행동을 실시간으로 생성하는 운동 능력을 담당함. [cite: 33, 39, 40]
* 이 모델을 통해 로봇은 "약 먹을 시간이야"와 같은 추상적인 명령을 듣고, 약병을 찾고 물을 가져오는 등의 절차를 스스로 추론하여 수행할 수 있음. [cite: 35, 36]
* 또한, 특정 명령 없이 저울과 토마토를 보고 무게를 재는 작업을 스스로 판단하는 등 상황을 이해하고 자율적으로 행동하는 능력을 목표로 함. [cite: 37, 38]

* 휴머노이드 AI를 학습시키기 위해 '데이터 팩토리'에서 대규모 데이터를 수집함. [cite: 44]
* 데이터 수집은 주로 사람이 원격으로 로봇을 조종하여 특정 작업을 반복 수행하는 방식으로 이루어지며, 이때 로봇이 보는 이미지, 손의 감각, 관절 각도 변화 등이 기록됨. [cite: 45, 46, 48]
* 이러한 데이터는 로봇의 '경험'에 해당하며, 성공과 실패 경험을 적절한 비율로 학습시켜 성능을 향상시킴. [cite: 46, 47]
* 최종 목표는 로봇이 스스로 데이터를 수집, 정제, 학습하여 성능을 높이는 '자율 성장' 단계를 구현하는 것으로, 이를 통해 새로운 수요가 발생했을 때 로봇이 스스로 훈련하여 작업에 투입될 수 있도록 함. [cite: 44, 56]
